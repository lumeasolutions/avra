BONNES PRATIQUES APPLIQUÉES
================================================================================

✓ React.memo sur tous les composants
  → Optimisation automatique des re-renders inutiles

✓ Hooks personnalisés pour la logique métier
  → useFacturation, useIAStudio, useStatistiques, etc.
  → Logique réutilisable et testable

✓ Utilitaires centralisés (lib/utils.ts)
  → Formatters (fmt, fmtPrecise)
  → Configurations (INVOICE_STATUS_CFG, DEVIS_STATUS_CFG)
  → Calculs (calcLignes)

  → Imports simplifiés: import { Component } from './components'
  → Facilite la réorganisation future

✓ Types TypeScript préservés
  → Tous les types originaux conservés
  → Props typées, store typé
  → Zero type errors

================================================================================
NOTES IMPORTANTES
================================================================================

1. Chaque composant est memoizé avec React.memo()
   → Empêche les re-renders inutiles
   → Performance optimisée automatiquement

2. Les pages refactorisées ne font qu'orchestrer
   → Logique = hooks
   → Présentation = composants
   → Orchestration = page

3. Tous les types TypeScript sont préservés
   → Pas de dégradation de typage
   → Imports depuis @/store intacts

4. Barrel exports pour des imports propres
   → import { Component } from './components'
   → Plus simple que import { Component } from './components/Component'

5. Réduction massive du code dans les pages
   → ~91% de réduction de code
   → Amélioration majeure de la maintenabilité
